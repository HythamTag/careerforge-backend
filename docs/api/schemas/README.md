# API Schemas

Complete schema documentation for all data structures used in the CV Enhancer Backend API.

---

## Schema Files

### Core Resources

1. **[User Schema](user-schema.md)** - User accounts, authentication, profiles, and statistics
2. **[Resume Schema](resume-schema.md)** - Resume containers, metadata, and file management
3. **[Version Schema](version-schema.md)** - Resume versions and content structure
4. **[Job Schema](job-schema.md)** - Background jobs, status, and execution logs

### Operations

5. **[Parsed Data Schema](parsed-data-schema.md)** - CV parsing results and extracted data
6. **[ATS Analysis Schema](ats-analysis-schema.md)** - ATS analysis results, scores, and recommendations
7. **[Enhancement Schema](enhancement-schema.md)** - AI enhancement results and templates

### Supporting Resources

8. **[Template Schema](template-schema.md)** - CV template configurations and metadata
9. **[Webhook Schema](webhook-schema.md)** - Webhook configurations, deliveries, and events
10. **[Subscription Schema](subscription-schema.md)** - User subscription plans and features

### Common Structures

11. **[Common Schemas](common-schemas.md)** - Success/error responses, pagination, HATEOAS links, and shared structures

---

## Schema Organization

Each schema file contains:

- **Object Definitions** - Complete structure with all fields
- **Request Schemas** - Request body structures
- **Response Schemas** - Response body structures
- **Field Descriptions** - Detailed field documentation with types and requirements
- **Examples** - JSON examples for each structure

---

## Usage

These schemas serve as:

- **Reference Documentation** - Complete data structure definitions
- **Validation Guide** - Field types, requirements, and constraints
- **Integration Guide** - For frontend developers and API consumers
- **Testing Reference** - For creating test data and mock responses
